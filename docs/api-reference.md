# API Reference — Rating Service

Base URL: `http://localhost:8084`

Write endpoints (`POST`, `PUT`, `DELETE`) require a user JWT:
```
Authorization: Bearer <access_token>
```

Read endpoints (`GET`) are publicly accessible — no token needed.

---

## Ratings — `/api/ratings`

### POST /api/ratings

Submit a rating for a movie.

**Authenticated** · Returns `201 Created`

A user may only rate a movie once. The movie must exist in the movie-service. After saving, the movie's `averageRating` is recalculated and pushed to the movie-service automatically.

**Request body**
```json
{
  "movieId": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "score": 4
}
```

**Validation rules**

| Field | Rules |
|-------|-------|
| `movieId` | Required, non-blank |
| `score` | Required, min 1, max 5 |

**Response**
```json
{
  "id": "7b3a1c92-4e8d-4f2a-b6c3-1d9e2f3a4b5c",
  "userId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "movieId": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "score": 4,
  "ratedAt": "2025-01-01T12:00:00"
}
```

**Errors**

| Status | Reason |
|--------|--------|
| 400 | Validation failure (missing fields, score out of range) |
| 400 | Movie not found in movie-service |
| 400 | User has already rated this movie |
| 401 | Missing or invalid JWT |
| 422 | Movie-service is unavailable |

### GET /api/ratings/movie/{movieId}

Get all ratings for a specific movie.

**Public** · Returns `200 OK`

**Response**
```json
[
  {
    "id": "7b3a1c92-4e8d-4f2a-b6c3-1d9e2f3a4b5c",
    "userId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "movieId": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
    "score": 4,
    "ratedAt": "2025-01-01T12:00:00"
  }
]
```

Returns an empty array if the movie has no ratings.

---

### GET /api/ratings/user/{userId}

Get all ratings submitted by a specific user.

**Public** · Returns `200 OK`

**Response** — same shape as `GET /api/ratings/movie/{movieId}`.

Returns an empty array if the user has no ratings.

---

### PUT /api/ratings/{id}

Update the score of an existing rating.

**Authenticated · Owner only** · Returns `200 OK`

Only the user who created the rating may update it. After saving, the movie's `averageRating` is recalculated automatically.

**Request body** — a single integer (the new score):
```json
5
```

**Validation rules**

| Rule | Description |
|------|-------------|
| Body required | Must not be null |

**Response** — updated `RatingResponse`.

**Errors**

| Status | Reason |
|--------|--------|
| 400 | Rating not found |
| 401 | Missing or invalid JWT |
| 403 | Authenticated but not the rating owner |

---

### DELETE /api/ratings/{id}

Delete a rating.

**Authenticated · Owner only** · Returns `204 No Content`

Only the user who created the rating may delete it. After deletion, the movie's `averageRating` is recalculated automatically.

**Errors**

| Status | Reason |
|--------|--------|
| 400 | Rating not found |
| 401 | Missing or invalid JWT |
| 403 | Authenticated but not the rating owner |

## Data Model

### Rating (Neo4j Node)

| Property | Type | Notes |
|----------|------|-------|
| `id` | String (UUID) | Generated by `UUIDStringGenerator` |
| `userId` | String (UUID) | ID of the user who submitted the rating |
| `movieId` | String (UUID) | ID of the rated movie in movie-service |
| `score` | Integer | 1–5 inclusive |
| `ratedAt` | LocalDateTime | Set automatically on save |

One rating per `(userId, movieId)` pair — enforced at the service layer.

## Common Error Responses

All errors follow [RFC 9457 Problem Details](https://www.rfc-editor.org/rfc/rfc9457):

```json
{
  "type": "about:blank",
  "title": "Bad Request",
  "status": 400,
  "detail": "You have already rated this movie",
  "instance": "/api/ratings"
}
```

Validation errors include a field-level `errors` map:

```json
{
  "type": "about:blank",
  "title": "Validation Error",
  "status": 400,
  "detail": "Validation failed",
  "errors": {
    "score": "Score must be at least 1",
    "movieId": "Movie ID is required"
  }
}
```

| Status | Meaning |
|--------|---------|
| 400 | Validation failure, resource not found, or duplicate rating |
| 401 | Missing or invalid JWT |
| 403 | Authenticated but not the resource owner |
| 422 | Upstream service unavailable (movie-service unreachable) |
| 500 | Unexpected server error |
